## Auth Pivot — OpenAlex Retired the Polite Pool

**This is the single most important fact to carry forward from this session.**

On 2026-02-13 OpenAlex retired the `mailto=` polite pool entirely. All API calls
now require a free API key passed as `api_key=<value>`. Anonymous access is
capped at 100 credits/day (effectively useless for real work). Free authenticated
access is $1.000/day in their new dollar-denominated display, equivalent to
~100,000 singleton calls.

**What this means:**

- Every future OpenAlex integration in Idiograph must load `OPENALEX_API_KEY`
  from `.env` via python-dotenv. The client in the spike is the reference pattern.
- The `mailto=` parameter is dead. It is not a fallback, not a courtesy, not a
  backup. It is silently ignored.
- The 10 rps hard rate limit still applies. Our 150ms sleep gives ~6–7 rps with
  comfortable margin. An asyncio token bucket becomes necessary only if we move
  to concurrent requests (deferred, tracked).
- Daily budget display is in dollars, not credits, on the settings page. The
  underlying quota is unchanged.

Memory should be updated to reflect this. The rate limiter work described in
prior memory (`asyncio token bucket or time.sleep(0.1)`) is partially
superseded: `time.sleep(0.150)` is the current correct value, the `mailto=` param
note is retired, and the `OPENALEX_API_KEY` requirement is new.

## Paste Truncation — Workflow Change

Long prompts truncated when pasted into Claude Code's input field mid-session.
Switched to file-based handoff:

1. Claude writes the prompt to a file.
2. User drops the file in the repo root or a `tmp/` directory.
3. Claude Code is told: "Read `prompt-name.md` and execute the instructions."

This also produces an audit trail of what was actually asked. Should become the
default for multi-step prompts going forward. `tmp/` and `*-prompt.md` patterns
added to `.gitignore` post-merge.
