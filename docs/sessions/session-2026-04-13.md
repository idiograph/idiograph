## What Was Decided

### Color Designer as Idiograph Domain
- Color Designer belongs in `domains/color_designer/` — not just `apps/color_designer/`
- The Qt canvas is a view. Handler implementations are pure Python. Executor owns execution.
- This is a stronger thesis statement: the color pipeline runs through the same executor as arXiv.

### Dependency Isolation
- `pyside6` added as optional extra `[qt]` in `pyproject.toml`
- arXiv server and Color Designer run as separate OS processes
- The SSE Drive node connection between them is HTTP — no shared process or memory

### Handler Registration
- `register_all()` demoted to test convenience only — never called from production entry points
- Each domain exposes its own explicit registration function
- arXiv: `register_arxiv_handlers()` in `domains/arxiv/handlers.py` — implemented
- Color Designer: `register_color_designer_handlers()` in `domains/color_designer/__init__.py` — pending

### Input Resolution — Content Shape (Option 2)
The `array_assign` handler receives two upstream inputs of different types (`color_array`
and `token_dict`). These are distinguished by inspecting the keys present in each upstream
result dict — duck typing over a closed, controlled vocabulary.

Edge label routing (Option 1) was discussed and explicitly deferred: arXiv does not need
it, and no current pipeline requires it. AMD note to be added: inputs-by-content-shape is
a known limitation; edge label routing is the eventual solution.

### Qt Node Translation Pattern
Each Qt node class implements `to_idiograph_node() -> Node`. The canvas implements
`build_graph() -> Graph`, walking its node list and wire system. The Save button in
`main.py` calls `canvas.build_graph()` then `asyncio.run(execute_graph(graph))`. The
handoff is one explicit line — legible, auditable, thesis-consistent.

### AMD-018 — Stable Node Identity
`BaseNode` had no stable identity. Two instances of the same type with identical params
were indistinguishable. `uuid4` assigned at construction as `self.node_id: str`.
Session-stable, never displayed, never persisted. Prerequisite for `to_idiograph_node()`
and `build_graph()`.

## Handler Designs — Confirmed

| Node type | params | returns |
|---|---|---|
| `color_swatch` | `hex`, `label` | `{color, label}` |
| `color_array` | `colors: [{label, hex}]` | `{color_array: [{label, hex}]}` |
| `schema` | `token_file` path | `{token_dict: {role: hex, ...}}` |
| `assign` | `role` | `{assignment: {role, hex}}` |
| `array_assign` | *(none)* | `{assignments: [{role, hex}, ...]}` |
| `write_tokens` | `token_file` path | `{status, count, path}` |

Input resolution for `array_assign`: content-shape inspection. Upstream result with
`color_array` key is the array input; upstream result with `token_dict` key is the
schema input.
